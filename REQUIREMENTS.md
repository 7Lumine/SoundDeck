# SoundDeck Requirements

## 目的

SoundDeck は、Windows 上で実マイク音声と MP3 音声をリアルタイムにミックスし、Discord などの VC アプリへ 1 つのマイク入力として渡すためのローカルツールです。

Discord へ直接音声を注入するのではなく、VB-CABLE などの仮想オーディオケーブルを出力先として使います。VC アプリ側では、その仮想ケーブルの録音側デバイスを入力デバイスとして選びます。

## 基本動作

通常時は、実マイクの音声をそのまま仮想ケーブルへ流します。

```text
実マイク
  -> SoundDeck
  -> CABLE Input
  -> CABLE Output
  -> Discord
```

MP3 再生中は、実マイク音声と 1 つ以上の MP3 音声を SoundDeck 内でミックスして仮想ケーブルへ流します。

```text
実マイク + MP3 A + MP3 B + ...
  -> SoundDeck
  -> CABLE Input
  -> CABLE Output
  -> Discord
```

Discord 側の入力デバイスは常に `CABLE Output` などの仮想ケーブル側に固定します。MP3 を流していない間も、ユーザーの声は通常のマイクのように相手へ届きます。

## 対象ユーザー

- Discord やゲーム内 VC に効果音、BGM、短い MP3 を流したいユーザー
- Soundpad のように、普段のマイク入力へ音声ファイルを重ねたいユーザー
- 自前の Windows 音声ドライバ開発までは行わず、既存の仮想オーディオケーブルを使って安全に実現したいユーザー

## 前提

- OS は Windows 10/11 を主対象とします。
- C#/.NET 製の Windows デスクトップアプリとして実装します。
- VC アプリに入力として見せるため、外部の仮想オーディオデバイスを使います。
- 最初の推奨構成は VB-CABLE とします。
- 自前の仮想マイクドライバ、APO、カーネルドライバは初期スコープに含めません。

## 必須要件

### デバイス選択

- 実マイク入力デバイスを選択できること。
- VC へ渡す出力デバイスを選択できること。
- 出力先には `CABLE Input` のような仮想ケーブルの再生側デバイスを選べること。
- デバイス一覧を再読み込みできること。
- 選択中デバイスが使えない場合は、エラー内容をユーザーに表示すること。

### マイクパススルー

- MP3 を再生していないときも、実マイク音声を出力先へ流し続けること。
- 遅延は VC 用途で違和感が少ない範囲に抑えること。
- マイク音量を調整できること。
- マイクを一時ミュートできること。

### MP3 再生

- MP3 ファイルを追加できること。
- 各 MP3 を個別に再生、停止できること。
- 複数の MP3 を同時に再生できること。
- 再生中の MP3 をまとめて停止できること。
- MP3 再生音量を調整できること。
- MP3 ごとの個別音量を調整できること。
- MP3 のループ再生を切り替えられること。
- MP3 再生中もマイク音声が途切れないこと。

### ミックス

- 出力音声は `実マイク + MP3` の合成音にすること。
- MP3 未再生時は `実マイクのみ` の合成結果にすること。
- 音割れを避けるため、ミックス後の音量をクリップしにくい形で処理すること。
- マイク音量と MP3 音量を別々に調整できること。

### VC 向け挙動

- Discord 側では入力デバイスを仮想ケーブルに固定したまま使えること。
- Push-to-Talk 利用時に、MP3 再生開始と同時にホットキーを押す運用を想定できること。
- Discord のノイズ抑制、エコー除去、自動ゲイン制御により音が変化する可能性を README に明記すること。

### UI

- 入力デバイス、出力デバイス、マイク音量、MP3 音量を 1 画面で確認できること。
- 現在の状態を `待機中`、`マイク送出中`、`MP3再生中`、`エラー` などで表示すること。
- MP3 リストを表示し、ダブルクリックまたはボタンで再生できること。
- 初回ユーザーでも「Discord では CABLE Output を選ぶ」と分かる表示を置くこと。

## 推奨要件

- MP3 再生中だけマイク音量を下げる ducking を切り替えられること。
- MP3 だけを流すための一時的なマイクミュートができること。
- 自分のヘッドホンへ MP3 をモニター再生できること。
- 効果音向けのグローバルホットキーを設定できること。
- 音量メーターでマイク入力、MP3、最終出力のレベルを確認できること。
- 設定を次回起動時に復元できること。

## 非要件

- Discord の内部 API へ音声を直接送ること。
- Discord アカウントのユーザートークンを使うこと。
- Windows に独自の仮想マイクデバイスを追加すること。
- Soundpad と同じドライバ拡張方式を初期版で再現すること。
- DRM 保護された音声や、VC アプリの制限を回避すること。

## 技術方針

初期実装では、ユーザーモードで扱える音声 API を使います。

- GUI: WPF
- デバイス I/O: NAudio
- MP3 読み込み: NAudio `AudioFileReader`
- ミックス処理: C# のリアルタイム音声処理
- 出力: 選択した Windows 再生デバイス

リアルタイム音声処理は、一定サイズの音声ブロック単位で行います。

```text
マイク入力ブロック
MP3再生ブロック
  -> 音量調整
  -> 加算ミックス
  -> クリップ防止
  -> 仮想ケーブルへ出力
```

## 初期版の完成条件

- 実マイクを選んで、仮想ケーブルへ常時送出できる。
- MP3 を再生すると、マイク音声に MP3 が重なって Discord 側へ届く。
- MP3 停止後、マイクのみの送出へ自然に戻る。
- マイク音量と MP3 音量を別々に調整できる。
- Discord で使うための手順が README に書かれている。

## 想定する初期ユーザーフロー

1. VB-CABLE をインストールする。
2. SoundDeck を起動する。
3. 入力デバイスに普段使っているマイクを選ぶ。
4. 出力デバイスに `CABLE Input` を選ぶ。
5. Discord の入力デバイスに `CABLE Output` を選ぶ。
6. SoundDeck の送出を開始する。
7. 普段はそのまま話す。
8. 必要なタイミングで MP3 を再生する。
9. MP3 停止後は普通のマイク状態に戻る。
